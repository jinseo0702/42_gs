지금 내 gnl의경우 다읽어 버린다. 결국 null이 있어도 읽어버리기 때문에 문제가 생긴다.
"a\n-fdsafsfsd" 의 경우 내 strdup에서 i < len
i == 0; len은 1이 되서 a만 복사하게 된다.
ft_get_head 의 문제 부분
else if (i == 1 && re_frf == 0 && *backup[0] != '\n')
		i -= 1;
다 읽었는데 "a\0"경우 를 말한다. 이때 i가 2가된다면 malloc한 것보다 크게 보게 된다.
그래서 고친 부분은
get_next_line에서 중복 읽기를 방지
	if (!ft_new_line(backup[fd], 10))
		re_frf = ft_read_file(fd, &backup[fd]);
에서 if문을 추가 해주었고
ft_get_head부분에서 이차원 배열을 바라볼수 있도록 연산자 순위를 지켜줬다.
	if (i == 1 && (*backup)[0] == '\n')
		i -= 1;
	else if (i == 1 && re_frf == 0 && (*backup)[1] == '\0')
		i -= 1;
이렇게 되면 문제 없이 잘 출력되는것을 알수 있다.
